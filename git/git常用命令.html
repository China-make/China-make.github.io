1基本操作

1.1设置全局用户名

  git config --global user.name make
  git config --global user.email 82763591@qq.com
  git status

1.2 在本地创建git仓库

创建版本库：
  $ mkdir learngit
  $ cd learngit
  $ pwd
  /Users/michael/learngit

初始化仓库：
  git init

添加文件到仓库：
  $ git add file1.txt
  $ git add file2.txt file3.txt
  $ git commit -m "add 3 files."

1.3 版本回退

git status                # 查看修改过的文件
git diff readme.txt       # 查看修改后的内容
git log                   # 显示从最近到最远的提交日志
git log --pretty=oneline  # 更清晰的看到最近提交的日志（也可以看到commit id）
git reset --hard HEAD^    # 回退到上一个版本（回退之后最新的版本就看不到了，所以提前纪录commit id）
用$ git reset --hard HEAD^回退到add distributed版本时，再想恢复到append GPL，就必须找到append GPL的commit id

git reflog                # 用来纪录每一次操作命令

1.4 拉取分支并切换分支

git fetch                 # 相当于是从远程获取最新版本到本地:

git branch -r             # 显示远程主机上的分支

git checkout -b linux origin/linux    #创建本地分支，本把远程分支放倒本地建立连接:

git checkout master       # 切换分支

git config --global core.editor "atom --wait"   #  设置atom为默认编辑器

1.5 撤销修改:

1、没有git add到暂存分区(工作分区中)
  git status
  git checkout -- file.txt  #把file.txt文件在工作区的修改全部撤销
  cat file.txt              #查看文件
2、git add到缓存分区了
  git status                #查看一下，修改只是添加到了暂存区，还没有提交
  git reset HEAD file.txt   #把暂存区的修改撤销
  git status
  git checkout -- file.txt  #同1相同
  git status

1.6 删除文件

git rm                    #删除文件
git commit                #提交
git checkout              #如果只是删除了而没有commit，则可以用该命令恢复

2 远程仓库与分支管理

2.1 git 推送到远程仓库

ssh-keygen -t rsa -C "youremail@example.com"  #-C 阐述详细信息
产生密钥把公钥放倒github中

git remote add origin git@git.coding.net:make-make/test.git    #origin是默认仓库名
git push -u origin master   #推送内容
git push
